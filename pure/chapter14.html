1. 시스템 관리자 역할

시스템 관리자는 사용자 계정, 파일 시스템, 네트워크, 프로세스, 보안 등 시스템 전반을 안정적으로 운영하는 역할을 담당한다.

  • 사용자 및 그룹 관리
  • 권한 및 보안 설정
  • 프로세스·서비스 관리
  • 백업 및 복구
  • 시스템 로그 모니터링

2. 사용자와 그룹 관리

1) 사용자 계정 추가/변경/삭제

useradd username
passwd username
usermod -l 새이름 username
userdel username

2) 그룹 관리

groupadd devteam
groupmod -n 새그룹 devteam
groupdel devteam

3) 주요 파일

  • /etc/passwd — 사용자 정보
  • /etc/shadow — 암호 정보
  • /etc/group — 그룹 정보

3. 파일 시스템 관리

마운트 / 언마운트

mount /dev/sdb1 /mnt/data
umount /mnt/data

디스크 사용량 확인

df -h        # 파일 시스템 전체 사용량
du -sh *     # 디렉토리별 사용량

파일 시스템 점검

fsck /dev/sda1

스왑 관리

swapon -s
swapoff /swapfile

4. 프로세스와 서비스 관리

프로세스 확인

ps -ef
top

프로세스 종료

kill PID
kill -9 PID       # 강제 종료

서비스 관리(systemctl)

systemctl status sshd
systemctl start sshd
systemctl stop sshd
systemctl enable sshd

5. 백업과 복구

tar

  • c — 생성
  • x — 압축 해제
  • v — 진행 출력
  • f — 파일 지정
tar cvf backup.tar /home/user
tar xvf backup.tar

rsync

rsync -av /source /backup

6. 로그 관리

시스템 로그는 문제 진단과 보안 분석에 필수적이다.

  • /var/log/messages — 시스템 일반 로그
  • /var/log/auth.log — 인증·로그인 기록
  • /var/log/secure — 보안 관련

로그 확인

tail -f /var/log/messages

7. 네트워크 관리

IP 확인

ifconfig
ip addr

라우팅 확인

route -n
ip route

네트워크 테스트

ping 8.8.8.8
netstat -an
ss -tuln

요약

  • 시스템 관리는 계정·파일 시스템·서비스·네트워크를 포함한다.
  • 중요 파일: passwd, shadow, group, log 파일들.
  • 백업은 tar/rsync 등이 핵심 도구.
  • 시스템 로그 분석은 문제 해결에 필수이다.
  • mount/df/du/ps/systemctl 등은 관리자 필수 명령어들이다.
