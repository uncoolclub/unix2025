1. 프로세스의 개념과 종류

프로세스(Process)는 실행 중인 프로그램을 의미한다. 유닉스는 다중 작업 운영체제로서 여러 프로세스가 동시에 실행될 수 있으며, 각 프로세스는 고유한 식별자(PID)를 가진다.

프로세스의 특징

  • PID (Process ID) — 각 프로세스의 고유 식별번호
  • PPID (Parent Process ID) — 부모 프로세스의 PID
  • UID (User ID) — 프로세스를 실행한 사용자 ID
  • GID (Group ID) — 프로세스를 실행한 그룹 ID

프로세스의 상태

  • R — 실행 중 (Running)
  • S — 수면 상태 (Sleeping)
  • D — 디스크 대기 상태
  • T — 정지 상태 (Stopped)
  • Z — 좀비 상태 (Zombie)

프로세스 유형

프로세스 설명
데몬(daemon) 유닉스 커널에 의해 실행되는 프로세스로 특정 서비스 제공
부모(parent) 자식 프로세스를 만드는 프로세스
자식(child) 부모에 의해 생성된 프로세스
실행이 끝나면 부모 프로세스에 결과를 돌려주고 종료
고아(orphan) 실행도중에 부모 프로세스가 종료된 프로세스
고아프로세스는 1번 프로세스를 새로운 부모로 가진다
좀비(zombie) 부모프로세스가 종료처리를 하지 않은 프로세스
종료가 되었음에도 할당된 시스템 자원이 회수되지 않은 상태

프로세스 계층 구조

유닉스에서 모든 프로세스는 트리 구조를 형성한다. 시스템 부팅 시 생성되는 init 프로세스(PID 1)가 모든 프로세스의 최상위 부모가 된다.

2. 프로세스 관리 명령

ps 명령어

ps 명령어는 현재 실행 중인 프로세스의 정보를 출력한다. 다양한 옵션을 통해 원하는 형태로 프로세스 정보를 확인할 수 있다.

기본 문법

ps [옵션]

주요 옵션

  • -e — 모든 프로세스 출력
  • -f — 자세한 정보 출력 (full format)
  • -l — 긴 형식으로 출력 (long format)
  • -u 사용자 — 특정 사용자의 프로세스만 출력
  • -p PID — 특정 PID의 프로세스만 출력
  • -t 터미널 — 특정 터미널의 프로세스만 출력

사용 예시

ps                    # 현재 터미널의 프로세스 출력
ps -e                 # 모든 프로세스 출력
ps -ef                # 모든 프로세스를 자세한 형식으로 출력
ps -u root            # root 사용자의 프로세스만 출력

pgrep 명령어

프로세스 이름이나 다른 속성으로 프로세스를 검색하여 PID를 출력한다.

사용 예시

pgrep firefox         # firefox 프로세스의 PID 출력
pgrep -u root         # root 사용자의 모든 프로세스 PID 출력
pgrep -f "python script.py"  # 명령행에 "python script.py"가 포함된 프로세스 검색

kill 명령어

kill 명령어는 실행 중인 프로세스에 신호(signal)를 보내어 프로세스를 제어한다. 주로 프로세스를 종료할 때 사용한다.

기본 문법

kill [옵션] [신호] PID

주요 신호

  • TERM (15) — 정상 종료 신호 (기본값)
  • KILL (9) — 강제 종료 신호
  • HUP (1) — 재시작 신호
  • INT (2) — 인터럽트 신호 (Ctrl+C와 동일)
  • STOP (19) — 일시 정지 신호
  • CONT (18) — 계속 실행 신호

사용 예시

kill 1234             # PID 1234 프로세스에 TERM 신호 전송
kill -9 1234          # PID 1234 프로세스 강제 종료
kill -HUP 1234        # PID 1234 프로세스 재시작

pkill 명령어

프로세스 이름으로 프로세스를 종료할 수 있는 명령어이다.

사용 예시

pkill firefox         # firefox 프로세스 모두 종료
pkill -9 httpd        # httpd 프로세스 모두 강제 종료
pkill -u user1        # user1의 모든 프로세스 종료

prstat 명령어

Solaris 시스템에서 사용되는 프로세스 통계 정보를 실시간으로 출력하는 명령어이다. (Linux에서는 top 명령어와 유사한 기능)

사용 예시

prstat                # 프로세스 통계 정보 출력
prstat -u user1       # 특정 사용자의 프로세스만 출력
prstat 5              # 5초 간격으로 갱신

top 명령어

top 명령어는 시스템의 프로세스를 실시간으로 모니터링하는 도구이다. CPU 사용률, 메모리 사용량 등을 동적으로 확인할 수 있다.

기본 사용법

top [옵션]

주요 옵션

  • -d 초 — 화면 갱신 간격 설정
  • -n 횟수 — 지정 횟수만큼 갱신 후 종료
  • -p PID — 특정 프로세스만 모니터링
  • -u 사용자 — 특정 사용자의 프로세스만 표시

대화형 명령어

  • q — 종료
  • k — 프로세스 종료 (kill)
  • r — 프로세스 우선순위 변경 (renice)
  • M — 메모리 사용량 순으로 정렬
  • P — CPU 사용률 순으로 정렬
  • T — 실행 시간 순으로 정렬

3. 포그라운드와 백그라운드 프로세스

유닉스에서는 프로세스를 포그라운드(foreground)와 백그라운드(background)로 실행할 수 있다. 백그라운드 실행을 통해 여러 작업을 동시에 수행할 수 있다.

포그라운드 프로세스

터미널에서 직접 실행되어 사용자의 입력을 기다리는 프로세스이다. 한 번에 하나의 포그라운드 프로세스만 실행될 수 있다.

백그라운드 프로세스

터미널과 독립적으로 실행되는 프로세스로, 사용자가 다른 작업을 할 수 있게 한다.

백그라운드 실행

명령어 &               # 명령어를 백그라운드에서 실행
find / -name "*.txt" & # 파일 검색을 백그라운드에서 실행

작업 제어 명령어

  • jobs — 현재 작업 목록 출력
  • fg [작업번호] — 백그라운드 작업을 포그라운드로 전환
  • bg [작업번호] — 정지된 작업을 백그라운드에서 계속 실행
  • Ctrl+Z — 현재 실행 중인 프로세스 일시 정지
  • Ctrl+C — 현재 실행 중인 프로세스 종료

사용 예시

jobs                  # 작업 목록 확인
fg 1                  # 작업 번호 1을 포그라운드로 전환
bg 2                  # 작업 번호 2를 백그라운드에서 계속 실행

nohup 명령어

터미널이 종료되어도 프로세스가 계속 실행되도록 하는 명령어이다.

nohup 명령어 &        # 터미널 종료와 무관하게 백그라운드 실행
nohup ./script.sh &   # 스크립트를 nohup으로 백그라운드 실행

4. 사용자 정보 보기

who 명령어

현재 시스템에 로그인한 사용자 정보를 출력한다.

주요 옵션

  • -b — 시스템 부팅 시간 출력
  • -r — 현재 런레벨 출력
  • -u — 사용자별 유휴 시간 출력
who                   # 로그인한 사용자 목록
who am i              # 현재 사용자 정보만 출력
who -b                # 시스템 부팅 시간 출력

w 명령어

who보다 더 자세한 사용자 정보를 출력한다. 로드 평균, 실행 중인 명령 등을 포함한다. 

w                     # 사용자별 상세 정보
w username            # 특정 사용자 정보만 출력

id 명령어

사용자의 UID, GID 및 소속 그룹 정보를 출력한다.

id                    # 현재 사용자의 ID 정보
id username           # 특정 사용자의 ID 정보

users 명령어

현재 로그인한 사용자 이름만 간단히 출력한다.

users                 # 로그인한 사용자 이름 목록

finger 명령어

사용자의 상세 정보를 출력한다 (시스템에 따라 설치되어 있지 않을 수 있음). 

finger                # 모든 사용자 정보
finger username       # 특정 사용자 정보

last 명령어

사용자의 로그인 기록을 출력한다.

last                  # 최근 로그인 기록
last username         # 특정 사용자의 로그인 기록
last -n 10            # 최근 10개 기록만 출력

요약

  • 프로세스는 실행 중인 프로그램으로, 데몬, 부모, 자식, 고아, 좀비 등의 유형이 있다.
  • ps, pgrep, kill, pkill, prstat, top 명령어로 프로세스를 관리할 수 있다.
  • 포그라운드와 백그라운드 실행을 통해 효율적인 다중 작업이 가능하다.
  • 다양한 명령어를 통해 현재 시스템의 사용자 정보를 확인할 수 있다.
