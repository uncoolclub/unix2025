1. 셸의 기능과 종류

셸은 사용자와 커널 사이에서 명령을 해석하고 실행하는 인터페이스이다.

셸의 주요 기능

  • 명령 해석기 — 사용자의 입력을 해석하여 적절한 프로그램 실행
  • 프로그래밍 기능 — 반복 작업을 스크립트로 작성해 자동화
  • 환경설정 기능 — 초기화 파일(.profile, .cshrc 등)로 사용자 환경 구성

대표적인 셸 종류

Bourne Shell (sh)

  • 최초의 본격적인 셸
  • 대부분의 유닉스 시스템에 기본 제공
  • 기능이 단순하고 스크립트 기반으로 많이 사용됨

Korn Shell (ksh)

  • Bourne Shell 기반
  • 히스토리, alias 등 편의 기능 제공
  • 처리 속도가 빠르고 호환성 높음

C Shell (csh)

  • 문법이 C 언어와 유사
  • 히스토리, alias 등 편의 기능 강화
  • 일반 사용자들이 많이 사용

Bash Shell (bash)

  • sh 기반 + ksh + csh 기능 포함
  • 리눅스의 기본 셸

2. 콘 셸(ksh)의 주요 기능

기본 정보

  • 경로: /usr/bin/ksh
  • 프롬프트: 일반 사용자 $, root #

Alias

alias ll="ls -l"
unalias ll

히스토리

  • HZ 변수로 저장 개수 지정 (기본 100)
  • 로그아웃 시 ~/.sh_history 저장

명령

history
r         # 직전 명령 재실행

자동완성 기능

파일명 앞부분 입력 → ESC + ESC 또는 TAB 으로 자동완성

초기화 파일

  • /etc/profile — 시스템 초기 설정
  • ~/.profile — 프롬프트, PATH 등 사용자 환경 설정
  • ~/.kshrc — alias, set 옵션 등 설정

터미널 설정(stty)

stty erase ^H     # 백스페이스 설정
stty -a           # 모든 옵션 출력

3. C 셸(csh)의 주요 기능

기본 정보

  • 경로: /usr/bin/csh
  • 프롬프트: 일반 %, root #

환경 변수 설정

setenv EDITOR vi

셸 변수 설정

set prompt="% "     # 프롬프트 설정

Alias

alias ll "ls -l"
unalias ll

히스토리

  • set history=100 — 저장 개수
  • set savehist=100 — 로그아웃 후 .history에 저장

히스토리 명령

history              # 목록
history -r           # 역순 출력
history 10           # 최근 10개

명령 재실행

!ls      # 최근 ls 명령 실행
!23      # 23번 명령 실행

파일명 자동 완성

set filec 설정 → 파일명 자동완성 지원

환경 설정 파일

  • /etc/.login — 시스템 초기 환경 설정
  • ~/.cshrc — 셸 변수, alias, 히스토리 설정
  • ~/.login — 로그인 시 1회 실행 (환경 변수, 터미널 설정)
  • ~/.logout — 로그아웃 시 실행

설정 파일 적용

source ~/.cshrc

요약

  • 셸은 명령 해석기, 프로그래밍 기능, 환경 설정 기능을 제공한다.
  • 대표적인 셸: sh, ksh, csh, bash.
  • ksh: alias, 히스토리, 자동완성, 초기화 파일(.profile, .kshrc) 사용.
  • csh: C 언어 스타일 문법, filec 자동완성, 환경 파일(.cshrc, .login) 사용.
  • 두 셸 모두 사용자 환경을 초기화 파일로 관리하고 명령 자동화 기능 제공.
