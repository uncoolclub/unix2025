1. 파일 아카이브 (tar)

tar(Tape ARchive) 명령어는 여러 파일과 디렉토리를 하나의 아카이브 파일로 묶는 도구이다. 원래는 테이프 백업용으로 개발되었지만, 현재는 파일 묶기와 백업의 표준 도구로 사용된다.

기본 문법

tar [옵션] [아카이브파일] [파일/디렉토리...]

주요 옵션

  • -c — 새로운 아카이브 생성 (create)
  • -x — 아카이브에서 파일 추출 (extract)
  • -t — 아카이브 내용 목록 보기 (list)
  • -v — 상세 정보 출력 (verbose)
  • -f — 아카이브 파일명 지정 (file)
  • -z — gzip 압축/해제와 함께 사용
  • -j — bzip2 압축/해제와 함께 사용
  • -r — 아카이브에 파일 추가
  • -u — 변경된 파일만 추가 (update)

아카이브 생성

tar -cf archive.tar file1 file2 dir1    # 아카이브 생성
tar -cvf backup.tar /home/user          # 상세 정보와 함께 아카이브 생성
tar -czf archive.tar.gz dir1            # gzip 압축과 함께 아카이브 생성
tar -cjf archive.tar.bz2 dir1           # bzip2 압축과 함께 아카이브 생성

아카이브 추출

tar -xf archive.tar                     # 아카이브 추출
tar -xvf archive.tar                    # 상세 정보와 함께 추출
tar -xzf archive.tar.gz                 # gzip 압축 해제와 함께 추출
tar -xjf archive.tar.bz2                # bzip2 압축 해제와 함께 추출
tar -xf archive.tar -C /tmp             # 특정 디렉토리에 추출

아카이브 내용 확인

tar -tf archive.tar                     # 아카이브 내용 목록
tar -tvf archive.tar                    # 상세 정보와 함께 목록 출력
tar -tzf archive.tar.gz                 # gzip 압축 파일 내용 목록

기타 작업

tar -rf archive.tar newfile.txt         # 아카이브에 파일 추가
tar -uf archive.tar dir1                # 변경된 파일만 업데이트

2. 파일 아카이브와 압축 (jar)

jar(Java ARchive) 명령어는 Java 플랫폼에서 사용되는 아카이브 도구이다. ZIP 파일 형식을 기반으로 하며, Java 클래스 파일과 관련 리소스를 패키징할 때 주로 사용된다.

기본 문법

jar [옵션] [jar파일] [파일/디렉토리...]

주요 옵션

  • -c — 새로운 아카이브 생성 (create)
  • -x — 아카이브에서 파일 추출 (extract)
  • -t — 아카이브 내용 목록 보기 (list)
  • -v — 상세 정보 출력 (verbose)
  • -f — 아카이브 파일명 지정 (file)
  • -u — 아카이브 업데이트
  • -m — 매니페스트 파일 포함
  • -0 — 압축하지 않고 저장만

JAR 파일 생성

jar -cf myapp.jar *.class               # 클래스 파일들을 JAR로 생성
jar -cvf myapp.jar com/                 # 디렉토리를 JAR로 생성 (상세 출력)
jar -cvfm myapp.jar manifest.txt *.class # 매니페스트 파일과 함께 생성

JAR 파일 추출

jar -xf myapp.jar                       # JAR 파일 추출
jar -xvf myapp.jar                      # 상세 정보와 함께 추출

JAR 파일 내용 확인

jar -tf myapp.jar                       # JAR 파일 내용 목록
jar -tvf myapp.jar                      # 상세 정보와 함께 목록 출력

JAR 파일 업데이트

jar -uf myapp.jar newfile.class         # JAR 파일에 새 파일 추가

3. .Z 파일 (compress / uncompress)

compress와 uncompress는 유닉스의 전통적인 압축 도구이다. LZW(Lempel-Ziv-Welch) 알고리즘을 사용하여 파일을 압축하며, 압축된 파일은 .Z 확장자를 가진다.

compress 명령어

파일을 압축하여 .Z 파일을 생성한다.

기본 문법

compress [옵션] 파일명

주요 옵션

  • -v — 압축 정보 출력 (verbose)
  • -c — 표준 출력으로 압축 결과 출력
  • -f — 강제 압축 (기존 파일 덮어쓰기)
  • -b 비트수 — 압축 비트 수 지정 (9-16, 기본값 16)

사용 예시

compress file.txt                       # file.txt를 file.txt.Z로 압축
compress -v file.txt                    # 압축 정보와 함께 압축
compress -c file.txt > file.txt.Z       # 원본 파일 유지하며 압축

uncompress 명령어

.Z 파일을 압축 해제한다.

기본 문법

uncompress [옵션] 파일명.Z

주요 옵션

  • -v — 압축 해제 정보 출력
  • -c — 표준 출력으로 압축 해제 결과 출력
  • -f — 강제 압축 해제

사용 예시

uncompress file.txt.Z                   # file.txt.Z를 file.txt로 압축 해제
uncompress -v file.txt.Z                # 압축 해제 정보와 함께 해제
uncompress -c file.txt.Z > file.txt     # 원본 .Z 파일 유지하며 압축 해제

4. .gz 파일 (gzip / gunzip)

gzip과 gunzip은 GNU에서 개발한 압축 도구이다. Deflate 알고리즘을 사용하여 compress보다 더 효율적인 압축을 제공하며, 압축된 파일은 .gz 확장자를 가진다.

gzip 명령어

파일을 압축하여 .gz 파일을 생성한다.

기본 문법

gzip [옵션] 파일명

주요 옵션

  • -v — 압축 정보 출력 (verbose)
  • -c — 표준 출력으로 압축 결과 출력
  • -d — 압축 해제 (decompress)
  • -f — 강제 압축
  • -k — 원본 파일 유지
  • -r — 디렉토리 내 파일들을 재귀적으로 압축
  • -1 ~ -9 — 압축 레벨 (1: 빠름, 9: 최대 압축)

사용 예시

gzip file.txt                           # file.txt를 file.txt.gz로 압축
gzip -v file.txt                        # 압축 정보와 함께 압축
gzip -c file.txt > file.txt.gz          # 원본 파일 유지하며 압축
gzip -9 file.txt                        # 최대 압축률로 압축
gzip -r directory/                      # 디렉토리 내 파일들을 재귀적으로 압축

gunzip 명령어

.gz 파일을 압축 해제한다.

기본 문법

gunzip [옵션] 파일명.gz

주요 옵션

  • -v — 압축 해제 정보 출력
  • -c — 표준 출력으로 압축 해제 결과 출력
  • -f — 강제 압축 해제
  • -k — 원본 .gz 파일 유지
  • -r — 디렉토리 내 .gz 파일들을 재귀적으로 압축 해제

사용 예시

gunzip file.txt.gz                      # file.txt.gz를 file.txt로 압축 해제
gunzip -v file.txt.gz                   # 압축 해제 정보와 함께 해제
gunzip -c file.txt.gz > file.txt        # 원본 .gz 파일 유지하며 압축 해제
gzip -d file.txt.gz                     # gzip -d로도 압축 해제 가능

zcat 명령어

압축된 파일의 내용을 압축 해제하지 않고 직접 출력한다.

zcat file.txt.gz                        # 압축 파일 내용을 화면에 출력
zcat file.txt.gz | grep "pattern"       # 압축 파일에서 패턴 검색

5. .zip 파일 (zip / unzip)

zip과 unzip은 널리 사용되는 압축 도구로, Windows와 호환되는 ZIP 형식을 지원한다. 여러 파일과 디렉토리를 하나의 압축 파일로 묶을 수 있다.

zip 명령어

파일과 디렉토리를 ZIP 형식으로 압축한다.

기본 문법

zip [옵션] 압축파일명.zip 파일/디렉토리...

주요 옵션

  • -r — 디렉토리를 재귀적으로 압축
  • -v — 상세 정보 출력
  • -q — 조용한 모드 (출력 최소화)
  • -u — 변경된 파일만 업데이트
  • -f — 새로운 파일만 추가
  • -d — 압축 파일에서 파일 삭제
  • -1 ~ -9 — 압축 레벨 (1: 빠름, 9: 최대 압축)
  • -x 패턴 — 특정 패턴의 파일 제외

사용 예시

zip archive.zip file1.txt file2.txt     # 파일들을 ZIP으로 압축
zip -r backup.zip directory/            # 디렉토리를 재귀적으로 압축
zip -v archive.zip *.txt                # 상세 정보와 함께 압축
zip -9 archive.zip largefile.txt        # 최대 압축률로 압축
zip -r archive.zip . -x "*.tmp"         # .tmp 파일 제외하고 압축

unzip 명령어

ZIP 파일을 압축 해제한다.

기본 문법

unzip [옵션] 압축파일명.zip

주요 옵션

  • -l — 압축 파일 내용 목록만 출력
  • -v — 상세 정보와 함께 목록 출력
  • -t — 압축 파일 무결성 테스트
  • -d 디렉토리 — 지정된 디렉토리에 압축 해제
  • -o — 기존 파일 덮어쓰기 (확인 없이)
  • -n — 기존 파일 덮어쓰지 않음
  • -j — 디렉토리 구조 무시하고 평면적으로 압축 해제
  • -q — 조용한 모드

사용 예시

unzip archive.zip                       # ZIP 파일 압축 해제
unzip -l archive.zip                    # ZIP 파일 내용 목록만 출력
unzip -d /tmp archive.zip               # /tmp 디렉토리에 압축 해제
unzip -o archive.zip                    # 기존 파일 덮어쓰기
unzip -t archive.zip                    # ZIP 파일 무결성 검사

6. .bz2 파일 (bzip2 / bunzip2)

bzip2와 bunzip2는 Burrows-Wheeler 알고리즘을 사용하는 고효율 압축 도구이다. gzip보다 더 높은 압축률을 제공하지만 압축/해제 속도는 상대적으로 느리다.

bzip2 명령어

파일을 압축하여 .bz2 파일을 생성한다.

기본 문법

bzip2 [옵션] 파일명

주요 옵션

  • -v — 압축 정보 출력 (verbose)
  • -c — 표준 출력으로 압축 결과 출력
  • -d — 압축 해제 (decompress)
  • -f — 강제 압축
  • -k — 원본 파일 유지
  • -t — 압축 파일 무결성 테스트
  • -1 ~ -9 — 압축 레벨 (1: 빠름, 9: 최대 압축, 기본값 9)

사용 예시

bzip2 file.txt                          # file.txt를 file.txt.bz2로 압축
bzip2 -v file.txt                       # 압축 정보와 함께 압축
bzip2 -c file.txt > file.txt.bz2        # 원본 파일 유지하며 압축
bzip2 -1 file.txt                       # 빠른 압축 (낮은 압축률)

bunzip2 명령어

.bz2 파일을 압축 해제한다.

기본 문법

bunzip2 [옵션] 파일명.bz2

주요 옵션

  • -v — 압축 해제 정보 출력
  • -c — 표준 출력으로 압축 해제 결과 출력
  • -f — 강제 압축 해제
  • -k — 원본 .bz2 파일 유지
  • -t — 압축 파일 무결성 테스트

사용 예시

bunzip2 file.txt.bz2                    # file.txt.bz2를 file.txt로 압축 해제
bunzip2 -v file.txt.bz2                 # 압축 해제 정보와 함께 해제
bunzip2 -c file.txt.bz2 > file.txt      # 원본 .bz2 파일 유지하며 압축 해제
bzip2 -d file.txt.bz2                   # bzip2 -d로도 압축 해제 가능

bzcat 명령어

압축된 파일의 내용을 압축 해제하지 않고 직접 출력한다.

bzcat file.txt.bz2                      # 압축 파일 내용을 화면에 출력
bzcat file.txt.bz2 | grep "pattern"     # 압축 파일에서 패턴 검색

압축률과 속도 비교

압축 도구 확장자 압축률 속도 특징
compress .Z 낮음 빠름 유닉스 전통 도구
gzip .gz 중간 빠름 가장 널리 사용
bzip2 .bz2 높음 느림 높은 압축률
zip .zip 중간 중간 Windows 호환

요약

  • tar는 파일 아카이브의 표준 도구로, 압축과 함께 사용할 수 있다.
  • jar는 Java 플랫폼의 아카이브 도구로 ZIP 형식을 기반으로 한다.
  • compress/uncompress는 전통적인 유닉스 압축 도구이다.
  • gzip/gunzip은 가장 널리 사용되는 압축 도구로 빠르고 효율적이다.
  • zip/unzip은 Windows와 호환되는 범용 압축 형식이다.
  • bzip2/bunzip2는 높은 압축률을 제공하는 고효율 압축 도구이다.
