1. 사용자간 직접 통신

유닉스 시스템에서는 같은 시스템에 로그인한 사용자들 간에 실시간으로 메시지를 주고받을 수 있는 다양한 통신 명령어를 제공한다.

write 명령어

write 명령어는 현재 로그인한 다른 사용자에게 실시간으로 메시지를 보내는 명령어이다.

기본 문법

write 사용자명 [터미널]

사용 방법

  • 명령어 실행 후 메시지를 입력하고 Enter
  • 메시지 전송을 종료하려면 Ctrl+D 또는 Ctrl+C
  • 상대방이 여러 터미널에 로그인한 경우 터미널 지정 가능

사용 예시

write john                              # john 사용자에게 메시지 전송
write mary tty1                         # mary 사용자의 tty1 터미널로 메시지 전송

wall 명령어

wall(write all) 명령어는 현재 로그인한 모든 사용자에게 동시에 메시지를 전송한다. 주로 시스템 관리자가 공지사항을 전달할 때 사용한다.

기본 문법

wall [메시지]
wall < 파일명

사용 예시

wall "시스템 점검으로 인해 10분 후 재부팅됩니다."
wall < notice.txt                       # 파일 내용을 모든 사용자에게 전송

talk 명령어

talk 명령어는 두 사용자 간의 실시간 대화를 가능하게 하는 대화형 통신 도구이다. 화면이 두 부분으로 나뉘어 양방향 대화가 가능하다.

기본 문법

talk 사용자명 [호스트명]

사용 방법

  • 상대방이 talk 요청을 수락하면 대화 시작
  • 화면 상단은 자신의 메시지, 하단은 상대방 메시지
  • 대화 종료는 Ctrl+C

사용 예시

talk alice                              # alice 사용자와 대화 요청
talk bob@server.com                     # 원격 호스트의 bob과 대화 요청

mesg 명령어

mesg 명령어는 다른 사용자로부터의 메시지 수신 여부를 제어한다. 메시지 수신을 허용하거나 차단할 수 있다.

기본 문법

mesg [y|n]

옵션

  • y — 메시지 수신 허용
  • n — 메시지 수신 차단
  • 옵션 없이 실행 — 현재 상태 확인

사용 예시

mesg                                    # 현재 메시지 수신 상태 확인
mesg y                                  # 메시지 수신 허용
mesg n                                  # 메시지 수신 차단

2. 이메일 보내기 (mailx)

mailx 명령어는 유닉스 시스템에서 이메일을 보내고 받는 명령행 기반 메일 클라이언트이다. 시스템 내부 사용자뿐만 아니라 외부 이메일 주소로도 메일을 보낼 수 있다.

기본 문법

mailx [옵션] [수신자...]

주요 옵션

  • -s 제목 — 이메일 제목 지정
  • -c 참조 — 참조(CC) 수신자 지정
  • -b 숨은참조 — 숨은참조(BCC) 수신자 지정
  • -a 첨부파일 — 파일 첨부
  • -r 발신자 — 발신자 주소 지정

이메일 보내기

대화형 모드

mailx user@example.com                 # 대화형으로 이메일 작성
Subject: 안녕하세요                      # 제목 입력
안녕하세요. 테스트 메일입니다.            # 본문 입력
.                                       # 점(.)으로 메일 전송

명령행에서 직접 전송

echo "메시지 내용" | mailx -s "제목" user@example.com
mailx -s "보고서" manager@company.com < report.txt
cat message.txt | mailx -s "공지사항" team@company.com

첨부파일과 함께 전송

mailx -s "문서 전송" -a document.pdf user@example.com < message.txt

이메일 읽기

mailx를 인자 없이 실행하면 받은 메일함을 확인할 수 있다.

메일 읽기 명령어

  • mailx — 메일함 열기
  • h — 메일 목록 보기
  • 1, 2, ... — 해당 번호의 메일 읽기
  • d 번호 — 메일 삭제
  • r — 답장
  • q — 종료

사용 예시

mailx                                   # 메일함 열기
& h                                     # 메일 목록 보기
& 1                                     # 첫 번째 메일 읽기
& d 1                                   # 첫 번째 메일 삭제
& q                                     # 메일 프로그램 종료

3. 호스트와 개인 정보 확인

네트워크 환경에서 호스트 정보와 사용자 정보를 확인하는 다양한 명령어들이 있다.

hostname 명령어

현재 시스템의 호스트명을 확인하거나 설정한다.

사용 예시

hostname                                # 현재 호스트명 출력
hostname -f                             # 완전한 도메인명(FQDN) 출력
hostname -i                             # 호스트의 IP 주소 출력

whoami 명령어

현재 로그인한 사용자의 이름을 출력한다.

사용 예시

whoami                                  # 현재 사용자명 출력

ping 명령어

네트워크 연결 상태를 확인하기 위해 ICMP 패킷을 전송한다.

기본 문법

ping [옵션] 호스트명_또는_IP주소

주요 옵션

  • -c 횟수 — 지정된 횟수만큼 ping 전송
  • -i 간격 — ping 전송 간격 설정 (초)
  • -s 크기 — 패킷 크기 설정
  • -t TTL — TTL(Time To Live) 값 설정

사용 예시

ping google.com                         # google.com에 연속 ping
ping -c 5 8.8.8.8                      # 8.8.8.8에 5번 ping
ping -c 3 -i 2 localhost               # localhost에 2초 간격으로 3번 ping

nslookup 명령어

DNS 조회를 통해 도메인명과 IP 주소 간의 변환 정보를 확인한다.

사용 예시

nslookup google.com                     # google.com의 IP 주소 조회
nslookup 8.8.8.8                       # 8.8.8.8의 도메인명 조회

host 명령어

DNS 조회를 수행하는 간단한 명령어이다.

사용 예시

host google.com                         # google.com의 DNS 정보 조회
host 8.8.8.8                           # 역방향 DNS 조회

finger 명령어

사용자 정보를 조회하는 명령어이다. (시스템에 따라 설치되어 있지 않을 수 있음)

사용 예시

finger                                  # 현재 로그인한 모든 사용자 정보
finger username                         # 특정 사용자 정보
finger @hostname                        # 원격 호스트의 사용자 정보

4. 파일 송수신 (ftp)

ftp(File Transfer Protocol) 명령어는 네트워크를 통해 파일을 송수신하는 표준 프로토콜이다. 원격 서버와 파일을 주고받을 때 사용한다.

기본 문법

ftp [옵션] [호스트명]

주요 옵션

  • -i — 대화형 모드 비활성화
  • -n — 자동 로그인 비활성화
  • -v — 상세 정보 출력
  • -d — 디버그 모드

FTP 연결 및 로그인

ftp ftp.example.com                     # FTP 서버에 연결
Name: username                          # 사용자명 입력
Password: ********                      # 비밀번호 입력

주요 FTP 명령어

디렉토리 관련

  • pwd — 원격 서버의 현재 디렉토리 확인
  • lpwd — 로컬의 현재 디렉토리 확인
  • ls — 원격 서버의 파일 목록
  • !ls — 로컬의 파일 목록
  • cd 디렉토리 — 원격 서버의 디렉토리 변경
  • lcd 디렉토리 — 로컬 디렉토리 변경
  • mkdir 디렉토리 — 원격 서버에 디렉토리 생성
  • rmdir 디렉토리 — 원격 서버의 디렉토리 삭제

파일 전송

  • get 파일명 — 원격 서버에서 로컬로 파일 다운로드
  • put 파일명 — 로컬에서 원격 서버로 파일 업로드
  • mget 패턴 — 여러 파일 다운로드
  • mput 패턴 — 여러 파일 업로드
  • delete 파일명 — 원격 서버의 파일 삭제
  • rename 원본 새이름 — 원격 서버의 파일명 변경

전송 모드

  • ascii — 텍스트 파일 전송 모드
  • binary — 바이너리 파일 전송 모드
  • type — 현재 전송 모드 확인

기타 명령어

  • status — 연결 상태 확인
  • help — 도움말 보기
  • quit 또는 bye — FTP 종료

FTP 사용 예시

ftp ftp.example.com                     # FTP 서버 연결
ftp> ls                                 # 원격 서버 파일 목록
ftp> cd public                          # 원격 서버의 public 디렉토리로 이동
ftp> lcd /home/user/downloads           # 로컬의 downloads 디렉토리로 이동
ftp> binary                             # 바이너리 모드로 설정
ftp> get largefile.zip                  # largefile.zip 다운로드
ftp> put document.pdf                   # document.pdf 업로드
ftp> mget *.txt                         # 모든 .txt 파일 다운로드
ftp> delete oldfile.txt                 # oldfile.txt 삭제
ftp> quit                               # FTP 종료

익명 FTP

일부 FTP 서버는 익명 접속을 허용한다. 사용자명으로 anonymous를 사용하고 비밀번호로 이메일 주소를 입력한다. 

ftp ftp.gnu.org                         # GNU FTP 서버 연결
Name: anonymous                         # 익명 사용자
Password: user@example.com              # 이메일 주소 입력

보안 고려사항

  • FTP는 암호화되지 않은 프로토콜이므로 보안에 취약
  • 중요한 데이터 전송 시에는 SFTP나 SCP 사용 권장
  • 공개 네트워크에서는 FTP 사용 자제

요약

  • write, wall, talk, mesg 명령어로 사용자 간 직접 통신이 가능하다.
  • mailx 명령어를 통해 이메일을 보내고 받을 수 있다.
  • hostname, ping, nslookup 등으로 호스트와 네트워크 정보를 확인할 수 있다.
  • ftp 명령어를 사용하여 원격 서버와 파일을 송수신할 수 있다.
  • 네트워크 통신 시에는 보안을 고려하여 적절한 프로토콜을 선택해야 한다.
