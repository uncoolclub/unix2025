1. 파일의 속성

유닉스는 다중 사용자 시스템으로, 각 파일에는 접근을 제어하기 위한 속성과 권한이 있다. ls -l 명령을 통해 파일의 종류, 권한, 소유자, 그룹, 크기, 수정 시간을 확인할 수 있다.

파일의 기본 속성

  • 파일 종류 — 일반 파일, 디렉토리, 장치 파일, 링크, 파이프, 소켓 등
  • 접근 권한 — 읽기(r), 쓰기(w), 실행(x)
  • 하드 링크 수
  • 파일 소유자 및 그룹
  • 파일 크기 (byte)
  • 마지막 수정 일시

파일 종류의 표시 예시

  • - : 일반 파일
  • d : 디렉토리
  • b : 블록 장치 파일
  • c : 문자 장치 파일
  • l : 심볼릭 링크
  • p : 파이프
  • s : 소켓

2. 파일의 접근 권한

파일 접근 권한은 사용자가 해당 파일을 읽거나, 쓰거나, 실행할 수 있는지를 제어한다. 이는 유닉스의 기본적인 보안 기능으로, 사용자 유형별로 권한을 다르게 설정할 수 있다.

사용자 카테고리

  • 소유자 (u) — 파일을 생성한 사용자
  • 그룹 (g) — 동일 그룹에 속한 사용자들
  • 기타 사용자 (o) — 나머지 모든 사용자

접근 권한의 의미

  • r — 읽기(Read): 파일 내용 조회 또는 디렉토리 목록 보기
  • w — 쓰기(Write): 파일 수정 또는 디렉토리 내 파일 생성/삭제
  • x — 실행(Execute): 프로그램 실행 또는 디렉토리 진입

권한 표기 예시

-rw-r--r--   (소유자: rw-, 그룹: r--, 기타: r--)

3. 기호를 이용한 접근 권한 변경

파일의 소유자는 chmod 명령으로 접근 권한을 변경할 수 있다. 기호 모드(Symbolic mode)는 사용자 그룹과 연산자, 권한 문자를 조합하여 사용한다.

기호 모드 구성

  • 사용자 카테고리: u(소유자), g(그룹), o(기타), a(전체)
  • 연산자: +(권한 추가), -(권한 제거), =(지정)
  • 권한 문자: r, w, x

예시

$ chmod u+w file.txt      # 소유자에게 쓰기 권한 추가
$ chmod g+x file.txt      # 그룹에 실행 권한 추가
$ chmod o=r file.txt      # 기타 사용자에게 읽기만 허용
$ chmod a+x file.txt      # 모든 사용자에게 실행 권한 추가

[실습 6-1] 기호 모드로 권한 변경

  • 파일 생성 및 권한 확인 (ls -l)
  • u-w, g+wx 등의 명령으로 권한 수정
  • 디렉토리 내 권한 일괄 변경 시 -R 옵션 사용

4. 숫자를 이용한 접근 권한 변경

숫자 모드(8진수 모드)는 각 사용자 범주의 접근 권한을 숫자로 표현하여 한 번에 설정할 수 있다.

8진수 표현 규칙

기호 2진수 8진수 의미
rwx 111 7 읽기, 쓰기, 실행
rw- 110 6 읽기, 쓰기
r-x 101 5 읽기, 실행
r-- 100 4 읽기
--x 001 1 실행
--- 000 0 권한 없음

예시

$ chmod 755 script.sh    # rwxr-xr-x
$ chmod 644 memo.txt     # rw-r--r--
$ chmod 700 secret.txt   # rwx------

[실습 6-2] 숫자 모드 실습

  • 권한 확인 후 755, 700, 644 등으로 변경
  • 숫자 모드가 기호 모드보다 빠르고 직관적임

5. 기본 접근 권한 설정 (umask)

파일이나 디렉토리를 새로 생성할 때 자동으로 부여되는 기본 권한은 umask 값에 따라 결정된다.

기본 권한

  • 일반 파일: 666 (rw-rw-rw-)
  • 실행 파일/디렉토리: 777 (rwxrwxrwx)

umask의 개념

umask는 “허용하지 않을 권한”을 지정하는 값이다. 기본 권한에서 umask 값을 뺀 결과가 실제 권한이 된다.

계산 예시

기본 권한: 666
umask: 022
결과: 644 (rw-r--r--)

기본 권한: 777
umask: 022
결과: 755 (rwxr-xr-x)

[실습 6-3] 기본 접근 권한 변경

  • umask 명령으로 현재 값 확인
  • umask 027 등으로 변경 후 새 파일 생성
  • 새 파일과 디렉토리의 기본 권한이 달라짐을 확인

요약

  • 파일의 속성과 접근 권한은 유닉스의 핵심 보안 기능이다.
  • 권한은 소유자, 그룹, 기타 사용자별로 r, w, x로 구분된다.
  • chmod 명령으로 기호 또는 숫자 모드로 권한을 변경할 수 있다.
  • umask를 이용해 새 파일의 기본 권한을 제어할 수 있다.
