1. 유닉스 편집기

편집기의 종류

  • 행 단위 편집기
    • ed — 유닉스 초기에 사용된 편집기, 현재는 거의 사용하지 않음
    • ex — 단독으로 사용되기보다는 vi와 함께 사용됨
  • 화면 단위 편집기
    • vim — vi의 확장판으로 현재 대부분의 유닉스 시스템에 기본 포함
    • Emacs — 강력한 기능을 제공하지만 별도 설치가 필요함

모드형과 비모드형 편집기

  • 모드형 — 명령모드와 입력모드를 구분해 사용. 명령은 명령모드에서, 텍스트 입력은 입력모드에서 수행함. (vim이 대표적)
  • 비모드형 — 모든 입력이 텍스트로 인식되며, 단축키 조합으로 명령을 실행. (예: 워드프로세서, 아래한글)

2. vim의 사용 방법

vim은 명령모드, 입력모드, 마지막행 모드의 세 가지 상태로 동작하는 모드형 편집기이다. 파일 작성, 수정, 저장, 검색 등 대부분의 작업을 키보드 명령으로 수행한다.

기본 동작

  • vim 파일명 — 새 파일 생성 또는 기존 파일 열기
  • :w — 저장
  • :q — 종료
  • :wq — 저장 후 종료
  • :q! — 변경 내용 무시 후 종료

모드 전환

  • 명령모드 → 입력모드: i, a, o
  • 입력모드 → 명령모드: Esc
  • 명령모드 → 마지막행 모드: :

커서 및 화면 이동

  • 좌우 이동: h(왼쪽), l(오른쪽)
  • 상하 이동: j(아래), k(위)
  • 행 단위 이동: 0(행 처음), $(행 끝)
  • 페이지 단위 이동: Ctrl+f(다음 화면), Ctrl+b(이전 화면)

편집 및 수정

  • x — 한 문자 삭제
  • dd — 현재 행 삭제
  • u — 이전 동작 취소 (Undo)
  • Ctrl+r — 취소한 동작 복구 (Redo)
  • yy — 현재 행 복사 / p — 붙여넣기

버퍼의 개념

  • 버퍼(Buffer)란 편집 중인 내용을 임시 저장하는 영역으로, 실행 취소나 복사/붙이기 동작에 사용된다.
  • "a3yy — 현재 행부터 3줄을 a버퍼에 저장
    "ap — a버퍼에 저장된 내용을 붙이기

검색 및 바꾸기

  • 검색: /단어 입력 후 Enter → n(다음 결과), N(이전 결과)
  • 치환: :%s/old/new/g — 파일 전체의 old를 new로 바꿈

비주얼 모드

  • v — 문자 단위 선택
  • V — 행 단위 선택
  • Ctrl+v — 블록(직사각형) 선택

기타 유용한 명령

  • :r 파일명 — 다른 파일 내용을 현재 문서로 읽어오기
  • :!명령 — vim 내부에서 셸 명령 실행
  • J — 두 행을 한 줄로 연결
  • . — 직전 명령을 반복 실행

실습 요약

  • [실습 4-1] 입력과 수정 — 파일 작성, 이동, 수정, 삭제, 복구, 저장
  • [실습 4-2] 편집 기능 — 복사/붙이기, 검색, 바꾸기
  • [실습 4-3] 기타 명령 — 여러 파일 다루기, 셸 명령 실행

3. vim의 환경 설정

vim은 다양한 설정 명령과 변수를 통해 사용자 환경을 제어할 수 있으며, 설정값은 ~/.vimrc 파일에 저장할 수 있다.

  • :set number — 행 번호 표시
  • :set list — 특수 문자 표시
  • :set showmode — 현재 모드 표시
  • :set all — 모든 환경 변수 표시

요약

  • 유닉스 편집기는 행 단위와 화면 단위로 구분되며, vim은 대표적인 화면 기반 편집기이다.
  • vim은 명령모드, 입력모드, 마지막행 모드로 구성된 모드형 편집기다.
  • 파일 편집, 검색, 버퍼 이용, 비주얼 모드 등 다양한 기능을 제공한다.
  • 환경 설정 명령을 통해 사용자 맞춤형 편집 환경을 구성할 수 있다.
