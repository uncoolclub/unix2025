1. 셸 기본 사용법 익히기

셸(Shell)은 사용자가 입력한 명령을 해석하고 실행하는 역할을 하는 프로그램이다. 커널과 사용자 사이에서 명령을 전달하며, 다양한 유틸리티와 환경 제어 기능을 제공한다.

셸의 종류와 특징

  • 대표적인 셸: sh(본 셸), ksh(콘 셸), csh(C 셸), bash(배시 셸)
  • 프롬프트 구분: $ (sh, ksh, bash) / % (csh)
  • 로그인 셸: 로그인 시 자동으로 실행되는 기본 셸
    서브셸: 사용자가 직접 실행한 셸 인스턴스

문자열 출력 명령

  • echo — 문자열을 출력하는 명령 (유틸리티/셸 내장 둘 다 가능)
  • print — 옵션을 제공하여 echo보다 유연하게 사용 가능

셸 특수 문자

  • * — 임의의 문자열
  • ? — 임의의 한 문자
  • [ ] — 대괄호 안의 문자 중 하나
  • ~ — 사용자의 홈 디렉토리
  • - — 이전 작업 디렉토리
  • ; — 여러 명령을 한 줄에 연결
  • | — 앞 명령의 출력을 뒤 명령의 입력으로 전달 (파이프)
  • ' ' — 내부 특수문자 해석하지 않음
  • " " — $, `, \를 제외하고 해석 안 함
  • ` ` — 명령 실행 결과를 문자열로 받음
  • \ — 특수문자의 의미 제거 (이스케이프)
  • >, >>, < — 입출력 방향 변경

2. 입출력 방향 변경

유닉스는 프로그램의 입력과 출력을 “표준 입출력 장치”로 처리하며, 리다이렉션(redirection)을 통해 파일이나 다른 명령으로 입출력을 전환할 수 있다.

표준 입출력 장치

  • 표준 입력(stdin): 키보드
  • 표준 출력(stdout): 화면
  • 표준 오류(stderr): 오류 메시지 출력용

리다이렉션 기호

  • 명령 > 파일 — 출력 결과를 파일에 저장 (덮어쓰기)
  • 명령 >> 파일 — 출력 결과를 파일 끝에 추가
  • 명령 2> 파일 — 오류 메시지를 파일로 저장
  • 명령 < 파일 — 입력을 키보드 대신 파일에서 읽음

3. 배시 셸의 환경 설정

환경 변수(Environment Variable)는 시스템이나 셸이 사용하는 설정값을 저장한다. bash에서는 셸 변수와 환경 변수를 구분해 사용한다.

변수 종류 및 확인

  • 셸 변수 — 현재 셸에서만 유효 (set 명령으로 확인)
  • 환경 변수 — 모든 셸에서 사용 가능 (env 명령으로 확인)
  • 변수 출력 — echo $변수명

변수 정의와 해제

  • 셸 변수 정의: 변수명=값
  • 환경 변수 정의: export 변수명=값 또는 변수명=값; export 변수명
  • 변수 해제: unset 변수명

자주 사용하는 Ctrl 키: Ctrl+C (명령 중단)

4. 배시 셸 명령 다루기

명령 단축 — alias

  • alias 별칭='명령' — 긴 명령이나 자주 쓰는 옵션을 짧게 지정
  • unalias 별칭 — 별칭 삭제
  • 인자를 전달해야 할 경우, 별칭 대신 함수(function)를 사용

명령 이력(history) 활용

  • fc — 이전 명령 목록 보기
  • ! — 직전 명령 또는 특정 명령 재실행
  • ↑ / ↓ — 이전 또는 다음 명령 불러오기
  • Esc + / — 명령 이력에서 문자열 검색 (vim 스타일 탐색)

5. 프롬프트 설정하기

프롬프트(prompt)는 명령 입력 대기 상태를 표시하며, PS1 환경 변수로 모양을 바꿀 수 있다.

기본 프롬프트 기호

  • $ — bash, sh, ksh
  • % — csh

PS1 설정 예시

  • PS1="Unix$ " — 문자열로 변경
  • PS1="\u@\h:\w\$ " — 사용자명@호스트:현재경로
  • PS1="\t \$ " — 시간 표시 추가

컬러 프롬프트 구성요소

  • \u — 사용자명 / \h — 호스트명
  • \w — 현재 디렉토리 / \t — 시각
  • \! — 명령 이력 번호
  • \e — 이스케이프 문자로 색상 제어

6. 환경 설정 파일 설정하기

로그인 시 자동 실행되는 설정 파일을 통해 사용자별 환경을 구성할 수 있다.

환경 설정 파일의 종류

  • 시스템 초기화 파일 — 관리자용 환경 초기화 (/etc/profile 등)
  • 사용자 초기화 파일 — 개인별 설정 (~/.profile, ~/.bashrc)

설정 파일 편집 및 적용

  • .bashrc — 앨리어스, 프롬프트, 경로 등 개인 설정 저장
  • source ~/.bashrc — 설정 파일 다시 실행 (변경 적용)
  • ~/.profile — 로그인 시 한 번만 실행되는 설정 파일

실습 요약

  • 홈 디렉토리 이동 후 .bashrc 수정
  • 프롬프트, 환경 변수, 앨리어스 설정 추가
  • 저장 후 source .bashrc 명령으로 실행

요약

  • 셸은 명령을 해석하고 실행하는 인터페이스로, bash는 대표적인 유닉스 셸이다.
  • 리다이렉션으로 입출력 방향을 자유롭게 변경할 수 있다.
  • 환경 변수와 alias를 통해 효율적이고 개인화된 환경을 구성할 수 있다.
  • PS1 설정으로 프롬프트 모양과 색상을 변경할 수 있다.
  • 환경 설정 파일( .bashrc, .profile )을 수정하여 자동화된 셸 환경을 만들 수 있다.
